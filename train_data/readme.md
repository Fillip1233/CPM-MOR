# train_data

---- GAR for mf-mor data ----
- mf_high_f.npy : 欧拉仿真的真实解 仿真时间步长: 200  数据量：200 端口数量：100 数据维度 (200,100,201)
- mf_in.npy : PULSE(V1 V2 TD TR TF PW PER) 数据维度（200,7） 200是数据量
- mf_low_f.npy : SVD-MOR 选取奇异值>2的低精度解 仿真时间步长: 200  数据量: 200 端口数量: 100 数据维度 (200,100,201)
- mf_inall.npy : PULSE输入 包含100个端口200个时间步的输入信息 数据量: 200 数据维度 (200,100,201)
- sim_100/ 100仿真步长, 各个端口输入一致的训练数据
- sim_100_diff/ 100仿真步长, 各个端口输入不一致的训练数据
-----------------------------
